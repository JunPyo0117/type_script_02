- OOP 객체지향 4대 특징

1. 추상화(Abstraction) > 클래스 interface
2. 캡슐화(encapsulation) > private, setter, getter
3. 상속(inherytance) > extends, abstract
4. 다형성(polymorphism) > extends, override

- OOP 객체지향 설계의 5원칙 SOLID

1. SRP(Single Responsibility Principle) 단일 책임 원칙
2. OCP(Open Closed Priciple) 개방 쳬쇄 원칙
3. LSP(Listov Substitution Priciple) 리스코프 치환원칙
4. ISP(Interface Segregation Principle) 인터페이스 분리 원칙
5. DIP(Dependency Inversion Principle) 의존 역전 원칙
